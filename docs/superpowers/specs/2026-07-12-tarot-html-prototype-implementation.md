## 1. 本轮目标

生成两个可以直接在浏览器打开的 HTML：

1. `assets/draw.html`：用户完成洗牌、选择三张牌、翻牌并复制抽牌码。
2. `assets/reading.html`：展示一次完整三牌解读的高保真示例报告。

本轮用于确认视觉方向和交互质感，不代表 78 张完整牌库已经完成。页面必须真实可操作、响应式、离线可打开，不能是静态效果图。

## 3. 图像语言

### 3.1 统一锚点

- 媒介：当代版画与蛋彩画结合，细密刻线、哑光矿物颜料、少量金属银墨质感。
- 时代：不出现现代物件，不直接复制 Rider-Waite-Smith 构图。
- 构图：竖版 2:3，主体完整，四周留稳定边框区域。
- 色彩：煤黑、骨白、氧化银、孔雀石绿，单张牌可有一个叙事辅色。
- 边框：统一的窄双线边框，不生成牌名、编号或任何文字，文字由 HTML 覆盖。
- 光线：像私人档案室中的侧光，安静、清晰、不过度阴森。
- 禁止：紫色星空、霓虹、外发光、玻璃质感、3D 渲染、照片感、已有塔罗品牌元素、水印、文字。

### 3.2 牌背

中心图形采用“闭合的双月门”：两个相对的新月围成竖直椭圆，内部是交错植物与星图刻线。图案必须上下近似对称，避免泄露正逆位。

### 3.3 纹理

- 桌面纹理没有明确物体，只呈现黑漆、细微擦痕和侧向微光，可无缝铺陈。
- 纸纹理没有文字、边框或污渍焦点，只呈现冷灰棉纸纤维，可无缝铺陈。

## 5. 抽牌页行为

### 5.1 状态

```text
ready -> shuffling -> drawing -> revealed -> code-ready
```

- `ready`：桌面中央显示牌堆和“开始洗牌”。
- `shuffling`：使用 transform/opacity 编排 1.4 秒，禁止点击。
- `drawing`：展示 8 张面朝下的牌，用户选择三张。
- `revealed`：所选牌移动到“现状、阻力、方向”三个稳定牌位并翻面。
- `code-ready`：生成 `TC-P1-...` 原型码，提供复制和手动选择回退。

### 5.2 随机

- 使用 `crypto.getRandomValues()` 完成 Fisher-Yates 洗牌。
- 原型牌组无放回选择。
- 每张牌独立产生正位或逆位。
- 浏览器不支持 Web Crypto 时停止流程并显示明确错误。

### 5.3 原型码

本轮使用可读原型格式：

```text
TC-P1-S3-00U.18R.09U-X7
```

- `P1`：prototype 1，避免与正式 `TC1` 协议混淆。
- `S3`：现状、阻力、方向三牌阵。
- `00U`：牌 ID 00，正位；`R` 表示逆位。
- `X7`：两位简单校验码，由完整前缀字符码求和后映射得到。

正式协议在 codec 阶段替换，不承诺兼容原型码。

## 6. 抽牌页布局

- 第一视口直接是牌桌，不出现产品介绍。
- 桌面：标题左上，状态右上，牌堆居中偏上，三个牌位位于下半部。
- 移动：标题和状态压缩成顶部一行，牌列横向可滚动，牌位保持三列。
- 卡牌统一 `aspect-ratio: 2 / 3`，选择、翻面和图片加载不改变布局。
- 所有主要控件最小点击区域 44px。

## 7. 报告页内容

使用固定示例数据验证真实排版：

- 问题：`我是否应该在近期改变工作方向？`
- 牌阵：现状、阻力、方向。
- 牌面：魔术师正位、月亮逆位、隐者正位。
- 内容：包含问题摘要、逐牌解读、牌间关系、两个行动建议、三个反思问题和边界说明。

报告是私人阅读档案，不使用聊天气泡、仪表盘统计、嵌套卡片或营销 CTA。

## 9. 动效与无障碍

- 仅动画 `transform` 和 `opacity`。
- `prefers-reduced-motion: reduce` 下跳过洗牌、扇开与翻转过渡。
- 键盘可操作全部牌背、确认和复制按钮。
- 牌背按钮使用 `aria-label="选择第 N 张牌"`，不提前暴露牌面。
- 正逆位同时通过旋转与文字表达。
- 状态变化使用 `aria-live="polite"`。
- 焦点样式使用孔雀石绿实线，不依赖外发光。
