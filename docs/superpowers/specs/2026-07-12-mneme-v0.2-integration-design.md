# v0.2 MNEME 本地记忆集成设计

- 日期：2026-07-12
- 状态：设计预留，不进入 v0.1 实现
- 依赖参考：`../MNEME/skills/mneme/SKILL.md`

## 1. 目标

把用户主动保存的塔罗记录和日记整理成一个本地、可读、可搜索、可删除的个人知识 bundle。Agent 在未来解读时可以引用长期主题，帮助用户发现变化和可行动方向，但不能把记忆包装成诊断或命运模型。

MNEME 的价值不只是向量检索。它的 L1 Markdown wiki 是用户和 Agent 都能检查的事实来源，L2 SQLite 向量索引只是可重建的派生层，dream 是保守的整理过程。

## 2. 集成原则

1. 默认关闭：未获得明确同意时，不保存任何对话、问题、牌面或报告。
2. 分次同意：保存本次阅读、保存日记、在未来解读中引用历史，是三个可独立撤回的动作。
3. 原文与总结分离：原始日记进入 `sources/`，长期主题写入概念页。
4. 可见可删：用户可以直接打开 Markdown，按日期查找并删除。
5. 索引可重建：`.mneme/index.db` 不是真相来源，删除原文和概念页后 reindex。
6. 不做心理画像：不记录“依恋型人格”“抑郁倾向”等推断标签。
7. 引用来源：使用历史信息时说明来自哪一篇日记或哪次阅读，不制造“我一直记得你”的拟人暗示。

## 3. 依赖方式

v0.2 首选“可选外部 Skill 适配”，不复制 MNEME 源码：

```text
tarot-confessional skill
        |
        | detect / invoke documented CLI
        v
mneme skill + local bundle
```

启动时按以下顺序发现：

1. 用户显式提供 `MNEME_BUNDLE`。
2. MNEME 自身配置文件。
3. 当前工作区向上发现带 `okf_version` 的 `index.md`。
4. 未找到时，询问是否创建独立的塔罗树洞 bundle。

塔罗 Skill 只依赖 MNEME 已公开的 `init`、`reindex`、`search` 和 host-agent ingest/dream 工作流，不直接读写 `.mneme/index.db`。

## 4. Bundle 建议结构

```text
tarot-journal/
├── index.md
├── log.md
├── sources/
│   ├── journal-2026-07-12.md
│   └── reading-2026-07-12-01.md
├── journal/
│   └── 2026-07-12-a-quiet-evening.md
├── readings/
│   └── 2026-07-12-career-direction.md
├── themes/
│   ├── boundaries.md
│   └── work-transition.md
└── reflections/
    └── 2026-07-week-2.md
```

推荐类型词：`journal-entry`、`tarot-reading`、`life-theme`、`reflection`。若 MNEME 类型词表未包含这些值，利用其“未知 type 允许但提示”的兼容策略，并在项目参考文档中声明。

## 5. 保存一次塔罗阅读

### 5.1 保存前

Agent 在报告完成后只问一次：

```text
要把这次阅读保存到你的本地树洞吗？会保存问题摘要、牌面、解读摘要和你愿意留下的感受，不保存完整聊天。
```

用户拒绝后不再次劝说。

### 5.2 原始记录

保存到 `sources/reading-<date>-<sequence>.md`：

```yaml
---
kind: tarot-reading-source
schema_version: "1"
created_at: 2026-07-12T20:30:00+08:00
spread_id: S3
draw_code: TC1-S3-...
consent: explicit
---
```

正文仅包含：问题摘要、牌面、Agent 解读、用户主动补充的感受。默认不保存完整对话。

### 5.3 概念页

MNEME ingest 将其整理为 `tarot-reading` 页面，包含：

- 日期和牌阵。
- 去身份化的问题摘要。
- 牌面与正逆位。
- 解读摘要。
- 用户认同或不认同的部分。
- 采取或准备采取的行动。
- 指向相关主题页的链接。

## 6. 日记与树洞

树洞模式首先是对话，不强迫结构化。只有用户要求“记下来”“写进日记”“以后提醒我”时才 ingest。

日记源文件保留用户原话，概念页可以做轻量整理，但不得改变事实和情绪立场。对高度私密内容，可以提供三种保存级别：

| 级别 | 保存内容 |
|---|---|
| 不保存 | 本次对话结束即止 |
| 只存摘要 | 保存用户确认过的去细节摘要 |
| 保存原文 | 原文进入本地 `sources/`，另建摘要页 |

## 7. 在新解读中使用历史

只有用户允许“参考过去记录”时执行：

```bash
python3 <mneme>/scripts/mneme.py search \
  "当前问题的核心主题" --json --type tarot-reading -k 5
```

Agent 应读取命中的概念页，而不是只相信向量片段。综合时使用以下顺序：

1. 当前用户表述。
2. 当前牌面和牌位。
3. 用户确认过的历史事实。
4. 历史主题的变化或重复。

报告中可写“你在 7 月 2 日的记录里也提到工作边界”，不能写“你一直都有边界问题”。前者有来源，后者是未经证实的性格判断。

## 8. dream 整理

dream 默认采用高保守度：先输出报告，用户确认后才改正式概念页。

允许的整理：

- 合并重复主题页。
- 为同一主题建立时间线。
- 发现已完成或已改变的行动。
- 汇总一周或一月反复出现的问题与资源。
- 建议值得回看的旧记录。

禁止的整理：

- 推断疾病、人格障碍、依恋类型或创伤诊断。
- 对用户或他人做善恶、危险性或可信度评分。
- 把塔罗牌出现频率当成统计学证据。
- 自动删除高敏感原始日记。
- 自动 push 到远端。

## 9. 删除、遗忘与导出

- `忘掉这次阅读`：删除对应 source、概念页、索引和 log 引用，再 reindex。
- `忘掉某个主题`：先列出受影响文件，用户确认后删除或去链接。
- `导出我的树洞`：直接打包 Markdown bundle，不依赖专有数据库格式。
- 删除操作完成后给出文件清单和验证结果，不只说“已经忘记”。

## 10. 安全事件

如果日记包含即时自伤或人身危险信息，Agent 优先处理当下安全，不先执行 dream、塔罗或长期模式分析。是否保存该内容必须重新询问，默认不保存危机细节。

## 11. v0.2 验收标准

- 未授权场景没有任何 bundle 写入。
- 只存摘要时，原始对话不进入 source。
- 搜索结果可追溯到具体 Markdown 文件。
- 删除后搜索不再返回对应内容。
- dream 失败时保留 pre-dream 快照，正式页面不出现半成品。
- dream 报告不包含心理诊断、人格标签或塔罗频率因果结论。
- MNEME 不存在时，v0.1 塔罗功能完全可用。
