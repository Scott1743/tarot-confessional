# 双 HTML 体验设计规范

- 日期：2026-07-12
- 状态：设计基线
- 页面：抽牌页 `draw.html`，解读报告 `reading.html`

## 1. Design Read

这是面向中文个人用户的沉浸式互动工具，不是落地页，也不是后台产品。视觉语言应是“夜间档案室里的私人牌桌”：高端、克制、神秘，但不使用廉价的紫色星空、霓虹光晕、满屏星座符号、玻璃卡片或金色装饰堆砌。

设计参数：

- `DESIGN_VARIANCE: 7`：桌面端有偏轴构图和空间层次，移动端收敛为单轴。
- `MOTION_INTENSITY: 7`：把动画集中在洗牌、选择、翻牌三个有意义的时刻。
- `VISUAL_DENSITY: 3`：抽牌页留白充分；报告页提高到 5 以支持阅读。

## 2. 视觉系统

### 2.1 核心意象

避免“宇宙占卜网站”，采用以下真实材质线索：

- 黑漆牌桌的微弱反光。
- 氧化银边线与冷灰纸张。
- 孔雀石绿作为唯一交互强调色。
- 旧版天文图谱的刻线逻辑，但只用于结构，不铺满装饰。
- 真实牌背、牌面和纸张纹理位图。

页面的记忆点是“呼吸的牌桌”：牌堆在用户确认问题后从暗处聚合，用户手势或指针只影响牌堆的轻微视差，所有能量集中在抽牌行为本身。

### 2.2 色彩

抽牌页使用固定暗色主题：

| Token | 色值 | 用途 |
|---|---|---|
| `night` | `#111310` | 页面背景 |
| `lacquer` | `#1B1E1A` | 牌桌和实体表面 |
| `paper` | `#E7E4DB` | 主文字和牌面纸色 |
| `ash` | `#A6A79F` | 次级文字 |
| `silver` | `#3E433D` | 结构线与边缘 |
| `malachite` | `#6F9E8C` | 唯一交互强调色 |

报告页使用冷灰纸张主题：

| Token | 色值 | 用途 |
|---|---|---|
| `folio` | `#E5E7E2` | 页面纸张 |
| `ink` | `#242724` | 正文 |
| `soft-ink` | `#646A64` | 注释 |
| `rule` | `#B9BDB7` | 分隔线 |
| `malachite` | `#527D6D` | 重点与链接 |
| `charcoal` | `#171A18` | 封面块和打印标题 |

同一页面不切换主题；两张页面作为不同阶段，可以分别使用夜色与纸色。

### 2.3 字体

- 标题：思源宋体或可再分发的 CJK 宋体 WOFF2 子集，强调文学感但不使用大段斜体。
- 正文：思源黑体或系统中文无衬线字体，确保小屏阅读。
- 抽牌码与数据：等宽字体 WOFF2 子集。
- 所有字体本地打包，`font-display: swap`，无远程字体请求。

### 2.4 图像资产

- 一张原创或许可明确的高分辨率牌背图。
- 78 张许可明确的牌面图，优先公共领域版本；发布时附 `ASSET_LICENSES.md`。
- 一张低对比度黑漆/纸纤维纹理位图，作为固定伪元素，避免滚动区域 GPU 重绘。
- 不手绘装饰 SVG，不使用不明来源的网络图片。

## 3. 抽牌页

### 3.1 页面任务

让用户完成一次专注、选择和抽牌，并可靠地把结果带回 Agent。首屏就是牌桌，不出现产品介绍、功能卡片或营销文案。

### 3.2 状态机

```text
READY -> SHUFFLING -> DRAWING -> REVEALED -> CODE_READY
              \-> ERROR
```

| 状态 | 用户看到 | 允许操作 |
|---|---|---|
| `READY` | 安静牌堆、问题标题、开始按钮 | 开始洗牌 |
| `SHUFFLING` | 一次 1.2-1.8 秒编排动画 | 减少动作模式下立即完成 |
| `DRAWING` | 扇形牌列和空牌位 | 选择指定数量的牌 |
| `REVEALED` | 牌翻面、牌位和正逆位 | 查看结果、确认 |
| `CODE_READY` | 抽牌码和复制命令 | 复制、重新复制，不默认重抽 |
| `ERROR` | 具体错误和恢复动作 | 重试或换浏览器 |

### 3.3 桌面构图

```text
+------------------------------------------------------+
|  问题标题                         牌阵 / 三张          |
|                                                      |
|                    [ 牌 堆 ]                         |
|             /  /  /  /  /  /  /                    |
|                                                      |
|      [现状空位]    [阻力空位]    [方向空位]           |
|                                                      |
|                    [开始 / 确认]                     |
+------------------------------------------------------+
```

标题靠左上，牌桌中心略偏右，三个牌位沿不完全对称的弧线排列。所有固定格式元素用 `aspect-ratio` 和明确的最小/最大尺寸，翻牌前后不改变布局。

### 3.4 移动构图

- 牌堆位于上半屏，牌位横向滚动或紧凑三列，不要求用户横屏。
- 主按钮固定在安全区上方，但不得遮挡牌位或抽牌码。
- 使用 `min-height: 100dvh`，适配浏览器地址栏变化。
- 最小点击区域 44 x 44 CSS px。

### 3.5 动效

| 动效 | 表达内容 | 实现 |
|---|---|---|
| 牌堆聚合 | 仪式开始、注意力集中 | `transform` + `opacity` |
| 洗牌 | 随机过程已发生 | CSS keyframes 或 Web Animations API |
| 选牌抬起 | 用户选择反馈 | translateY、rotate、shadow |
| 翻牌 | 状态由未知变为已知 | 3D rotateY，固定容器尺寸 |
| 码出现 | 流程完成 | 一次淡入，不循环闪烁 |

不使用滚动劫持、无限漂浮、鼠标追光或自定义光标。所有动画在 `prefers-reduced-motion: reduce` 下变为即时状态切换。

### 3.6 文案

文案是命令和状态，不解释产品功能：

- `把问题留在心里。`
- `开始洗牌`
- `选择一张` / `选择三张`
- `这次抽牌已经完成`
- `复制抽牌码`
- `回到对话，把抽牌码发给我`

错误文案必须说明恢复方式，例如：`无法使用安全随机数。请用较新的 Chrome、Safari、Edge 或 Firefox 打开此页。`

## 4. 解读报告页

### 4.1 页面任务

把 Agent 的解读整理成一份私人阅读档案。它不是数据仪表盘，也不是聊天记录截图。

### 4.2 信息结构

1. 封面：标题、日期、牌阵，不展示用户姓名。
2. 问题摘要：一段经去隐私处理的简短文字。
3. 牌阵全景：牌图、牌位、正逆位关系。
4. 逐牌解读：每张牌一节，图像与文字交替但不机械重复。
5. 核心答复与综合判断：先回应用户原问题，再说明牌与牌之间的联系、关键条件和不确定性。
6. 可行动方向：最多两项。
7. 留给自己的问题：最多三项。
8. 边界说明：低调但可读。

### 4.3 桌面构图

```text
+------------------------------------------------------+
| 私人阅读档案                                          |
| 关于下一步的三张牌                                    |
|                         [日期 / S3]                   |
|------------------------------------------------------|
| 问题摘要                                               |
|                                                      |
| [牌1]-----------[牌2]-----------[牌3]                 |
| 现状             阻力             方向                |
|                                                      |
| 解读正文，以纵向轴线串联牌位                           |
|                                                      |
| 可行动方向                  留给自己的问题             |
+------------------------------------------------------+
```

避免把每张牌塞入同样的圆角卡片。用纸张层级、细线和图文节奏组织内容，圆角最多 4px。

### 4.4 打印与保存

- 提供 `@media print`，隐藏复制/打印按钮。
- A4 页面避免牌图和标题跨页断开。
- 保持白底或浅灰底，节约墨水且确保 PDF 可读。
- 所有图像带尺寸，避免布局偏移。
- 页面无网络依赖，保存 HTML 后仍可打开。

## 5. 无障碍与质量门槛

- 正文对比度达到 WCAG AA，目标 AAA。
- 所有交互有可见焦点，键盘顺序与视觉顺序一致。
- 牌背按钮有可访问名称，如“选择第 4 张牌”，不提前泄露牌面。
- 正逆位不只依赖旋转，文本同时标明。
- 动态状态使用克制的 `aria-live` 提示。
- 360 x 800、768 x 1024、1440 x 900 三个视口做截图验收。
- 用 Playwright 检查无重叠、无横向溢出、复制回退、重载行为和打印布局。
- 视觉验收必须看真实牌图与真实中文长文本，不能只看占位符。

## 6. 需要用户最终确认的视觉决策

在实现前提供两张高保真方向稿供选择，但底层交互不变：

1. `夜间档案室`：黑漆、氧化银、孔雀石，当前推荐方向。
2. `冷月剧场`：冷灰幕布、硬光牌面、更具舞台感，动效更强。

不提供紫色星空或金色神殿方向，除非品牌后续明确要求。
