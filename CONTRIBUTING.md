# Contributing

感谢你参与改进塔罗树洞。

## 提交改动

1. Fork 仓库并从 `main` 创建短生命周期分支。
2. 保持改动聚焦，并同步更新相关文档。
3. 验证普通占卜、纯倾诉和高风险请求三类场景。
4. 运行 `git diff --check`。
5. 提交 Pull Request，说明问题、方案、验证方法和用户可见变化。

## 内容要求

- 解读应保留不确定性，不宣称预言必然发生。
- 不添加歧视、恐吓、操控或诱导依赖的内容。
- 不提交真实用户的私密倾诉、身份信息或对话记录。
- 安全漏洞不要通过公开 Issue 披露，请参照 `SECURITY.md`。

## Issue

报告问题时，请提供可复现的用户提示、实际行为、期望行为和运行平台。示例应使用虚构信息并移除隐私数据。
